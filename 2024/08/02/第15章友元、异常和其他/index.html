15.1 友元

  • 类并非只能拥有友元函数，也可以将类作为友元。在这种情况下，友元类的所有方法都可以访问原始类的私有成员和保护成员。 p602
  • 例如：假定需要编写一个模拟电视机和遥控器的简单程序。决定定义一个Tv类和一个 Remote 类，来分别表示电视机和遥控器。遥控器并非电视机，反之亦然，所以公有继承的is-a关系并不适用。遥控器也非电视机的一部分，反之亦然，因此包含或私有继承和保护继承的 has-a 关系也不适用。事实上，遥控器可以改变电视机的状态，这表明应将 Remote 类作为 Tv 类的一个友元。 p602
  • 如果不使用友元类，则必须将Tv类的private成员设置为public，但是这样会出现同一个遥控器可用于多台电视机，这是不符合常理的。 p606
  • 友元
    • 友元类：类作为友元
    • 友元成员函数：类的成员函数做友元
    • 其他友元和类的组合形式：两个类互相为对方的友元类
    • 共同的友元：函数需要访问两个类的私有数据，在两个类中将该函数作为友元。

15.3 异常

处理异常的方式： p616

  • 使用abort()函数，如果在函数中调用它，会直接退出程序，不会返回main函数

  • 使用函数返回值返回错误码

  • 使用异常机制，异常的处理有3个组成部分：

    • 引发异常，使用throw关键字抛出异常
    • 使用处理程序(exception handler)捕获异常，catch关键字表示捕获异常
    • 使用try块，try块标识在块中的代码可能引发异常

  • 异常类型可以是字符串或其他C++类型；通常为类类型。 p620

使用异常机制的例子： p619

调用hmean(10, -10)引发异常后，抛出一个异常，程序返回到try块的末尾。将执行catch块，将异常字符串赋给s，catch块中程序执行到continue时，返回到while循环的开始位置。

  • **栈解退(unwinding the stack)：**假设现在函数由于出现异常而终止，则程序也将释放栈中的内存，但不会在释放栈中的第一个地址后停止，而是继续寻找，在找到try块之前，栈中的内存会被释放掉，直到找到一个位于try块中的返回地址，。程序跳转到try块的末尾，而不是返回到函数调用后的下一条语句。(throw与return的区别就在此) p625

  • 如果没有栈解退这种特性，则引发异常后，对于中间函数调用放在栈中的自动类对象，其析构函数将不会被调用。 p625

  • 虽然 throw-catch 机制类似于函数参数和函数返回机制，但还是有些不同之处： p629

    • 其中之一是函数 fun()中的返回语句将控制权返回到调用 fun()的函数，但 throw 语句将控制权向上返回到第一个这样的函数：包含能够捕获相应异常的 try-catch 组合。
    • 另一个不同之处是，引发异常时编译器总是创建一个临时拷贝，即使异常规范和catch块中指定的是引用。因为抛出异常的函数执行完后，throw的类对象将不存在，这也是为什么编译器要创建一个临时拷贝。catch块使用引用是因为基类引用可以执行派生类对象。

  • C++提供了很多基于exception的异常类型 p631

    • stdexcept头文件中包含了处理的异常类
    • bad_alloc异常和new：bad_alloc异常类位于头文件new中，让new引发bad_alloc异常
    • 空指针和new，C++标准提供了一种在失败时返回空指针的new，语法：int *p = new (std::nothorw) int;

  • 要捕获所有异常(不管是预期的异常还是意外异常)，可以这样做：

    • 包含头文件#include <exception>，using声明using namespace std;

    • 将意外异常操作指定为调用该函数：set_unexpected(myUnexpected);

15.4 RTTI

RTTI是运行阶段类型识别(Runtime Type Identification)的简称。 p642

C++ Primer Plus P642

假设有一个类层次结构，其中的类都是从同一个基类派生而来的，则可以让基类指针指向其中任何一个类的对象。这样便可以调用这样的函数：在处理一些信息后，选择一个类，并创建这种类型的对象，然后返回它的地址，而该地址可以被赋给基类指针。如何知道指针指向的是哪种对象呢？

为何要知道类型。可能希望调用类方法的正确版本，在这种情况下，只要该函数是类层次结构中所有成员都拥有的虚函数，则并不真正需要知道对象的类型。但派生对象可能包含不是继承而来的方法，在这种情况下，只有某些类型的对象可以使用该方法。也可能是出于调试目的,想跟踪生成的对象的类型。对于后两种情况，RTTI提供解决方案。

简而言之，派生对象可能包含了不是继承而来的方法，需要知道其类型，RTTI就是解决这种情况的。

C++有以下3个组件支持RTTI： p643

  • dynamic_cast运算符，将基类指针指向派生类指针，否则运算符返回0(也就是空指针)
  • typeid运算符返回一个指出对象的类型的值
  • type_info结构存储了有关特定类型的信息

dynamic_cast语法：dynamic_cast<Type *>(pt)。

  • 如果(pt*)是Type或者Type直接或间接派生而来的类型，那么就表示将指针pt转换为Type类型的指针。也就是说这么写表示派生类向基类转换(向上转换)。
  • 也可以让基类指针转为派生类指针，如果转换失败，则会返回空指针

15.5 类型转换运算符

4个类型转换运算符： p650

  • dynamic_cast，既可以向上转换(本身就是安全的)，也可以向下转换(如果转换失败，返回空指针)。
  • const_cast，有时候可能需要这样一个值(这个值必须是非const类型，只是被const类型的指针指向了)，它在大多数时候是常量，而有时在这种情况下又是可以修改。因此，可以将这个值声明为 const，并在需要修改它的时候，使用const_cast。
  • static_cast语法：static_cast<type_name>(expression)只有当type_name可以被隐式转换为expression所属的类型或expression所属的类型可以被隐式转换为type_name，才能使用static_cast
  • reinterpret_cast，它不允许删除 const，但会执行其他令人生厌的操作。有时程序员必须做一些依赖于实现的、令人生厌的操作，使用reinterpret_cast运算符可以简化对这种行为的跟踪工作。

4种运算符语法：类型转换运算符<类型名称>(表达式)。除了static_cast可以将整型转换为枚举值、double转为int、将float转换为long以及其他各种数据值转换以外，其他3个类型转换运算符的类型名称只能是指针或引用，类型名称是指针时，表达式必须为指针；类型名称是引用时，表达式必须为引用。

15.6 总结

  • C++异常机制为处理拙劣的编程事件，如不适当的值、I/O 失败等，提供了一种灵活的方式。 p652
  • RTTI(运行阶段类型信息)特性让程序能够检测对象的类型。
  • 与通用转换机制相比，dynamic_cast、static_cast、const_cast和 reinterpret_cast 提供了更安全、更明确的类型转换。

2、您知道了如何建立相互类友元的方法。能够创建一种更为严格的友情关系，即类B只有部分成员是类 A 的友元，而类A只有部分成员是类B的友元吗？请解释原因。

不。为使类A拥有一个本身为类B的成员函数的友元，B的声明必须位于A的声明之前。一个前向声明是不够的，因为这种声明可以告诉 A：B是一个类；但它不能指出类成员的名称。同样，如果B拥有一个本身是A的成员函数的友元，则A的这个声明必须位于B的声明之前。这两个要求是互斥的。

4、throw和 return 之间的区别何在?

假设函数f1()调用函数f2()。

  • f2()中的return语句导致程序执行在函数f1()中调用函数f2()后面的一条语句。
  • throw 语句导致程序沿函数调用的当前序列回溯，直到找到直接或间接包含对f2()的调用的try语句块为止。它可能在f1()中、调用 f1()的函数中或其他函数中。找到这样的 ty语句块后，将执行下一个匹配的 catch 语句块，而不是函数调用后的语句。

7、static_cast运算符与 dynamic_cast 运算符有什么不同？

dynamic_cast 运算符只允许沿类层次结构向上转换(向下转换时具有类型检查的功能)，而static_cast运算符允许向上转换和向下转换。static_cast 运算符还允许枚举类型和整型之间以及数值类型之间的转换。
