18.1 复习前面介绍过的 C++11 功能

  • C++11新增了类型long long和unsigned long long，以支持64位(或更宽)的整型

  • 新增了支持16位字符的char16_t和支持32位字符char32_t的类型

  • 初始化列表语法可防止缩窄，即禁止将数值赋给无法存储它的数值变量。比如：

  • C++11提供了模板类initializer_list，可将其用作构造函数的参数。 p796

  • C++11提供了多种简化声明的功能

    • auto，自动类型推断
    • decltype，将变量的类型声明为表达式指定的类型。例如：
    • 返回类型后置，C++11新增一种函数声明语法：在函数名和参数列表后面指定返回类型： p798
    • 模板别名：using=
    • 空指针nullptr

  • 智能指针，C++11摈弃了auto_ptr，新增unique_ptr、shared_ptr和weak_ptr

  • 异常规范

  • 作用域枚举，使用作用域运算符来访问新形式的枚举New1::never、New2::never

  • 对类的修改

    • 显示转换关键字explicit，禁止单参数构造函数导致的自动转换
    • C++11运行在类中使用大括号或等号初始化成员

  • 左值(rvalue)与右值(rvalue)的区别 链接

    • 左值的地址是可以被访问的，即可以使用&运算符取得左值的地址
    • 右值是任何不是左值的东西，这意味着我们不能取得右值的地址，它也不会在单个表达式之外持续存在。

  • 右值引用，引用右值引用的主要目的之一是实现移动语义(move semantics)

    • 传统的C++引用(现在称为左值引用)使得标识符关联到左值
    • C++11新增了右值引用，使用&&来表示。右值引用可关联到右值，即可出现在赋值表式右边，但不能对其应用地址运算符的值。右值包括字面常量(C-风格字符串除外，它表示地址)、诸如x+y等表达式以及返回值的函数(条件是该函数返回的不是引用)：

    #1是调用复制构造函数，而#2是调用移动构造函数，函数返回值的临时变量不会被删除，而是将右值引用与allcaps()函数返回值关联到一起。

  • 但是，具体是否调用移动构造函数，还是由编译器决定，使用g++编译器与VS2010输出的结果就不同。编译器可进行某种优化，省略移动构造函数，只要结果与未优化时相同。 p808

  • 和复制赋值运算符类似，赋值运算符也可用于移动构造函数 p809

  • 强制移动：使用std::move()函数，该函数返回参数的右值引用。它在头文件utility中已经声明。 p810

18.3 新的类功能

  • 在原有4个特殊成员函数的基础上，C++11新增两个：移动构造函数、移动赋值运算符 p813
    • 默认构造函数(defalut constructor)
    • 复制构造函数(copy constructor)
    • 复制赋值运算符(copy assginment operator)
    • 析构函数(destructor)
    • 移动构造函数(move constructor)
    • 移动赋值运算符(move assginment operator)
  • 编如果您没有提供复制构造函数，而代码又需要使用它，编译器将提供一个默认的复制构造函数。如果您没有提供移动构造函数，而代码又需要使用它，编译器将提供一个默认的移动构造函数。
  • 委托构造函数(Delegating Constructors)：有些构造函数可能需要包含其他构造函数中已有的代码。为让编码工作更简单、更可靠，C++11允许在一个构造函数的定义中使用另一个构造函数。 p815
  • 继承构造函数，使用using命令，这让派生类继承基类的所有构造函数(默认构造函数、复制构造函数和移动构造函数除外)，但不会使用与派生类构造函数的特征相匹配(参数类型和个数相同)的构造函数:
  • 管理虚方法：override和final p817
    • 假设基类声明了一个虚方法，而您决定在派生类中提供不同的版本，这将覆盖旧版本。如果参数类型不匹配，将隐藏而不是覆盖旧版本。
    • C++11使用虚说明符override指出要覆盖一个虚函数。如果声明与基类方法不匹配，编译器将视为错误
    • 禁止派生类覆盖特定的虚方法，在参数列表后面加上final说明符
    • 说明符override和final并非关键字，而是具有特殊含义的标识符。在其他上下文中，可将它们用作常规标识符，如变量名或枚举

18.4 Lambda函数

  • lambda函数，也称lambda表达式，常简称为lambda
  • 比较函数指针、函数符和Lambda函数 p818
    • 可以使用lambda表达式替换函数指针或函数符构造函数
    • 仅当lambda表达式完全由一条返回语句组成时，自动类型推断才管用；否则，需要使用新增的返回类型后置语法：[](double x)->double{int y = x; return x - y;}，该lambda函数的返回值为double类型。
    • lambda表达式本身就是替代函数指针和函数对象，可以这么写：auto fun = [](double x)->double{int y = x; return x - y;}，fun是一个函数对象。
    • lambda表达式可访问作用域内的任何动态变量，要捕获要是使用的变量，可将其名称放在中括号内： p821
      • [z]将按值访问变量z；
      • [&count]将按引用访问变量count
      • [&]能够按引用访问所有的动态变量
      • [=]能够访问所有动态变量
      • 还可以混合使用，比如[ted,&ed]，能够按值访问ted，按照引用访问ed；[&, ted]表示按值访问ted以及按照引用访问其他所有动态变量
  • C++11中引入lambda的主要目的是，能够将类似于函数的表达式用作接受函数指针或函数符的函数的参数。因此，典型的lambda是测试表达式或比较表达式，可编写为一条返回语句。这使lambda 简洁而易于理解，且可自动推断返回类型。比如STL中的count_if函数的第三个参数，接受一元谓词(返回 bool 值的一元函数是谓词)，这种情况就可以使用lambda，如下：

18.5 包装器

  • C++11提供了多个包装器(wrapper，也叫是适配器[adapter])。这些对象用于给其他编程接口提供更一致或更合适的接口。 p823
  • 包装器function：头文件functional中声明了模板function。语法：std::function<double(char ,int)> fdci;创建一个名为fdci的function对象，它接受一个char参数和一个int参数，并返回一个double值。
